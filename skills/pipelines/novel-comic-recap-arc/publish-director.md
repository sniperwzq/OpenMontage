# Publish Director - 小说漫画解说 Arc Pipeline

## 目标

把完成后的 Arc recap 包装成适合短视频平台发布的素材，同时不泄露 protected future arcs。

## 发布 Metadata 规则

重点强调：

- 一个完整 mini-story。
- 本 Arc 核心的 betrayal / status reversal。
- 女主在本 Arc 结束时的 landing decision。
- 下一 Arc 的好奇心。

避免：

- 泄露 protected future arcs。
- 命名 Arc brief 中隐藏的 twists。
- 使用和实际画面不匹配的泛泛 romance-drama 标题。

## 输出要求

创建 schema-valid 的 `publish_log`，包含：

- final video path
- platform title options
- short description
- thumbnail concept
- hero frame recommendation
- content warnings，如果需要
- metadata 中的 arc id 和 source file references

## 标题模式

使用直接的西方短视频标题表达：

- `He Left His Luna To Die In The Snow`
- `Her Husband Chose The Other Woman First`
- `The Luna Finally Stopped Begging`

标题必须具体对应实际 Arc，不要套模板骗点击。

## 质量门

- Packaging 真实代表已完成视频。
- Thumbnail concept 来自实际 hero frame。
- Metadata 不泄露后续受保护故事内容。
